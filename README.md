# 🔐 Active Directory Authentication Monitoring (Wazuh SIEM)

## 📌 Objective
This project demonstrates monitoring and analysis of Windows authentication events in an Active Directory environment using Wazuh SIEM.  
The focus is on detecting failed login attempts and understanding how authentication data is represented in SIEM logs.

---

## 🧱 Lab Environment

| Component         | Configuration                            |
| ----------------- | ---------------------------------------- |
| Domain Controller | Windows Server (`lab.local`)             |
| Client Machine    | Windows 10 (Domain Joined)               |
| SIEM              | Wazuh                                    |
| Network           | VirtualBox Host-Only (`192.168.56.0/24`) |

### 🧪 Action Performed
Multiple failed login attempts were generated by entering incorrect credentials on a domain-joined Windows 10 system.

- `win.eventdata.WorkstationName`: `WIN10`

---
## 🧠 Analysis

The `TargetUserName` field appeared in **User Principal Name (UPN)** format:



This indicates that authentication attempts were made using **domain-qualified credentials** rather than the traditional `DOMAIN\username` format.

Although Wazuh did not normalize this into a top-level user field, the username was still present within the raw event data (`win.eventdata.TargetUserName`).

Multiple failed attempts within a short timeframe indicate potential **brute force activity**, a common attack technique targeting authentication systems.

---

## ⚠️ Troubleshooting Insights

During setup, several issues were encountered and resolved:

- Windows 10 initially on a different network (could not reach Domain Controller)  
- IP address conflict between client and server  
- Missing DNS configuration prevented domain resolution  

Resulting errors included:
- “DNS server not responding”  
- Domain join failures  

✅ Correcting DNS to point to the Domain Controller resolved authentication and detection issues.

---

## 🎯 Key Takeaways

- Active Directory relies heavily on proper DNS configuration  
- Wazuh can detect authentication failures but may not always normalize fields  
- Understanding raw Windows event logs is essential for accurate analysis  
- Failed login patterns (Event ID **4625**) are critical indicators of potential attacks
